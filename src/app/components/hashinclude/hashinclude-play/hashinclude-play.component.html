Problem

Uno is trying to solve a puzzle. The task is very simple. He is given 2 tables and is asked to merge the two after interchanging the rows and columns of the second table . The merging is to happen in such a way that the resultant table has as many number of rows and columns as there are in the individual tables. Also the entry in resultant table should be such that the letters are progressively interleaved and the elements are taken in alternate orders from each table.

Input Format

Line 1: Number of test cases
Line 2: Order of the matrices (mxm)
Line 3: row1 of matrix 1
Line 4: row2 of matrix 1
Line m+2: rowm of matrix 1
Line m+3: row1 of matrix 2
Line m+4: row2 of matrix 2
Line 2xm: rowm of matrix 2

Output Format

Row1
Row2
...
Rowm

Sample Input

1
3x3
hello hi there
how are you
abc def cfg
lmn opq rst
jpeg jpg doc
pdf psd ppt

Sample Output

hlelmnlo jhpeig tphedfre
ohpqow ajrepg pysdou
arbcst ddocef cpfgpt

Explanation

2nd matrix is transposed.
Starting from 1st row, 1st word of 1st matrix. Take first character from 1st matrix followed by 1st character of 2nd matrix. Then 2nd and 3rd char of 1st matrix and 2nd and 3rd char of 2nd matrix and so on.
As 1st word has been taken from 1st matrix, start second word from matrix 2 and 3rd from matrix 1 and so on(alternatively).
Also as 1st row is taken from 1st matrix, take 2nd from 2nd matrix and so on.

Problem

Romeo wants to give Juliet a garland. There are C different colors of flowers. There are Fi flowers of i-th color.
A garland can be created using the following algorithm.
1: Arrange all the flowers in any order such that flowers of the same color are placed together.
2: The garland initially consists of only the first flower from the arrangement.
3: For each subsequent flower in order, join it to a flower of the same color in the garland. If there is no flower of the same color, it can be joined to any flower in the garland.
All flowers are distinct, even if they have the same color. How many different garlands can be formed by following the above algorithm? Two Garlands are considered different if two flowers are joined by a thread in one configuration, but not in the other.
NOTE: Think of the FLOWER formation as a tree and not as a straight line. Any number of FLOWERS can be connected to a FLOWER.

Input Format

The first line contains the number of test cases T. Each test case contains C on the first line - the number of colors of flowers. The next line contains C integers, where the i-th integer Fi denotes the number of flowers of the color.

Output Format

Output T lines, one for each test case. All answers should be output modulo 1000000007 .

Constraints

1 <= T <= 20
1 <= C <= 1000000
1 <= Fi <= 30

Sample Input

5
2
2 1
2
2 2
1
4
2
3 1
5
1 1 1 1 1

Sample Output

2
4
16
9
125

Explanation


Testcase 1:

Let us label the Flowers A1,A2 and B1. Initially, they can be arranged in 4 ways - "A1,A2,B1", "A2,A1,B1", "B1,A1,A2", and "B1,A2,A1".
For each of the first two arrangements, an Garland can be formed in 2 ways (A1-A2-B1 or B1-A1-A2 from the first one and A2-A1-B1 or B1-A2-A1 from the second one).
For each of the last two arrangements, an Garland can be formed in 1 way.
However, of the total 6 possible Garlands, there are only 2 unique ones : A1 - A2 - B1, and A2 - A1 - B1.

Testcase 2:

The possible unique Garlands are A1 - A2 - B1 - B2, A1 - A2 - B2 - B1, A2 - A1 - B1 - B2, and A2 - A1 - B2 - B1.

Testcase 3:

For the third test-case, it might be easier to see there are only types of graphs on vertices: the path or the star. It's not hard to see that there are 12 paths and 4 stars

Testcase 5:

For the fifth test-case, a lot of people claimed that the total number of possible ways is 60. But that is wrong. The correct answer is 125. Here's the hint: Once again, you've to think of it as a tree.
So one possible arrangement can be:
A is a root node and has two edges (A-B and A-C). Now, think of B as a sub-root node with two edges (B-D and B-E). Similarly, you can figure out the other possible Flower arrangements. This will lead you to the correct answer.

Problem

There are N Classrooms in MEC. Each pair of classess are directly connected by a unique directed path, and each path has its own toll that must be paid every time it is used. You're planning a Visit to MEC, and it should satisfy the following conditions: You can start in any class. You must use 2 or more different paths (meaning you will visit 2 or more classes). At the end of your trip, you should be back in class where you began. The average cost (sum of tolls paid per path traveled) should be minimum. Can you calculate the minimum average cost of a visit to MEC.

Input Format

The first line is an integer, N (number of classes). The N subsequent lines of N space-separated integers each describe the respective tolls on moving from class i to class j ; in other words, the j-th integer of the i-th line denotes the toll for moving from class i to class j. Note: As there are no paths from one class to the same class, the i-th integer of line i will always be 0 .

Output Format

Print the minimum cost as a rational number p/q (tolls paid over paths moved). The greatest common divisor of p and q should be 1.

Constraints

1 < N < 500
0 < C <= 200
Paths Covered >= 2

Sample Input

2
0 1
2 0

Sample Output

3/2

Explanation

The toll from class C0 to class C1 is 1.
The toll from c1 to c0 is 2.
Your travel cost p = 1 + 2 = 3.
Your number of roads traveled is q = 2. Thus, we print 3/2 as our answer.
